Rhine

Haskell FRP with type-level clocks

About

Rhine is a clock-safe library for synchronous and asynchronous Functional Reactive Programming (FRP) in Haskell.

It separates the aspects of clocking, scheduling and resampling from each other, and ensures clock-safety at the type level.

Why Rhine

Coordinating systems running at different rates is a hard problem in general. Many kinds of bugs can occur, such as accidental synchronisation of independent subsystems, concurrency issues, buffer underruns and overflows, and space and time leaks.

Functional descriptions of reactive systems suffer from similar issues. Functional Reactive Programming frameworks either work synchronously, or leave it to users to coordinate subsystems. Other asynchronous reactive frameworks show similar limitations.

In Rhine, clock information is expressed at the type level, so that incorrectly clocked programs are rejected at compile time.

Ecosystem

  • rhine-bayes
    Backend to the monad-bayes library, enabling you to write stochastic processes as signal functions, and performing online machine learning on them.
  • rhine-terminal
    Backend to the haskell-terminal library, enabling you to write terminal applications as signal functions.
  • rhine-gloss
    Backend to the gloss library, enabling you to write gloss applications as signal functions.
  • rhine-sdl2
    Backend to the SDL2 library, enabling you to write graphical applications as signal functions. Some example games are provided.
  • rhine-linux
    Rhine frontends for various Linux subsystems such as D-Bus and inotify, enabling you to write system applications.
  • rhine-chat
    Rhine frontends for various chat servers such as Discord and Matrix, enabling you to write chat clients and bots as signal functions.
